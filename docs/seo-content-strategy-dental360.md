## 🔗 Aturan Internal Linking

### 1. CLUSTER → PILLAR (WAJIB)
- Setiap cluster article **HARUS** link ke Pillar-nya
- Anchor text: nama pillar atau variasi natural
- Posisi: di paragraf pembuka dan/atau penutup

**Contoh:**
```
Dalam artikel [Software Manajemen Klinik Gigi: Panduan Lengkap],
kami sudah bahas bagaimana sistem digital bisa mengubah praktik Anda.
```

### 2. PILLAR → CLUSTER (OPSIONAL)
- Pillar boleh link ke cluster yang relevan
- Gunakan untuk deep-dive topik tertentu

### 3. CLUSTER ↔ CLUSTER (JIKA RELEVAN)
- Cluster dalam pillar yang sama saling link
- Contoh: "WhatsApp Otomatis" link ke "Reminder Appointment"

**Contoh:**
```
Selain WhatsApp otomatis, Anda juga bisa memanfaatkan
[reminder appointment pasien] untuk mengurangi no-show.
```

### 4. CROSS-PILLAR (JIKA SANGAT RELEVAN)
- Cluster dari pillar berbeda boleh link
- Contoh: "Dashboard Bisnis" link ke "Pillar 1 Software"

### 5. SEMUA ARTIKEL → LANDING PAGE DENTAL360
- CTA di akhir artikel mengarah ke `/dental360`
- Gunakan UTM tracking untuk mengukur konversi

**Contoh CTA:**
```
---
Tertarik mengimplementasikan sistem manajemen klinik gigi?
[Coba Dental360 sekarang] dan rasakan perbedaannya!
```

## ✅ Checklist SEO per Artikel

### On-Page SEO

| Elemen | Deskripsi | Checklist |
|--------|-----------|-----------|
| **Title Tag** | Mengandung keyword utama di awal, max 60 karakter | [ ] |
| **Meta Description** | Mengandung keyword, CTA, max 155 karakter | [ ] |
| **URL Slug** | Pendek, mengandung keyword, gunakan tanda hubung | [ ] |
| **H1** | Sama dengan title, mengandung keyword | [ ] |
| **H2/H3** | Subheading dengan variasi keyword dan LSI | [ ] |
| **Keyword Density** | 1-2% dari total kata | [ ] |
| **Internal Links** | Minimal 3-5 link per artikel | [ ] |
| **External Links** | 1-2 link ke sumber autoritatif | [ ] |
| **Images** | Alt text mengandung keyword, compressed | [ ] |
| **CTA** | Arah ke Dental360 di akhir artikel | [ ] |

### Struktur Artikel Ideal

```
1. HOOK (100-150 kata)
   - Pertanyaan atau statement menarik
   - Introduce masalah
   - Link ke Pillar (jika cluster article)

2. INTI ARTIKEL (80% dari total kata)
   - H2 untuk setiap sub-topik
   - Bullet points dan numbered lists
   - Data/statistik untuk kredibilitas
   - Internal links ke artikel relevan

3. PENUTUP & CTA (150-200 kata)
   - Ringkasan poin utama
   - CTA ke Dental360
   - Link ke artikel terkait
```

## 🛠️ Cara Menggunakan Dokumen Ini

### Workflow per Artikel:

```
1. PILIH ARTIKEL
   ↓ Lihat "DAFTAR ARTIKEL" di atas, cari yang statusnya ⬜

2. UPDATE STATUS → 🔄 (saat mulai kerjakan)
   ↓ Edit file ini, ubah status jadi 🔄

3. TULIS ARTIKEL
   ↓ Buat file di src/content/blog/[slug].md
   ↓ Ikuti template & checklist SEO

4. UPDATE STATUS → ✅ (saat selesai)
   ↓ Edit file ini:
     - Ubah status jadi ✅
     - Isi tanggal selesai
     - Isi nama file yang dibuat
   ↓ Update "Progress Overview" di atas
```

### Format Update Progress:

Setelah selesai 1 artikel, update:
1. **Progress Overview** (di paling atas) - tambah angka selesai
2. **DAFTAR ARTIKEL** - ubah status, isi file
3. **Jadwal Rilis** - ubah status, isi tanggal & file
